# Dogfood Report: Foundry SimpleVault Fixture

## Repo / Protocol

- Protocol: SimpleVault
- Repository: checked-in fixture at `safeharbor-cli/examples/foundry-simple-vault`
- Commit: local working tree fixture
- Foundry config: `safeharbor-cli/examples/foundry-simple-vault/foundry.toml`
- SafeHarbor config: generated by `safeharbor-cli/scripts/verify-sample-workflow.sh`
- Operator: Phase 8 release hardening
- Date: 2026-04-15

## Commands Run

```sh
bash safeharbor-cli/scripts/verify-sample-workflow.sh
```

The script runs:

```sh
shcli scan --config safeharbor.toml
shcli review --config safeharbor.toml --approve-defaults
shcli compile --config safeharbor.toml
shcli battlechain prepare --config safeharbor.toml
shcli status --config safeharbor.toml
shcli doctor --config safeharbor.toml
shcli registry publish --config safeharbor.toml --manifest-uri ipfs://bafy-safeharbor-sample
```

It also runs the `agent-sdk` `simple_consumer` example against the compiled manifest.

## What Worked

- Scan produced analysis graph, structural candidates, and standards recognition using the mock Aderyn fixture.
- Non-interactive review produced review state and reviewed input.
- Compile produced final manifest JSON and summary markdown.
- BattleChain prepare produced `.safeharbor/battlechain/prepare.json`.
- Status and doctor completed locally without live RPC.
- Registry publish produced `.safeharbor/registry/publish.json` without signing or network access.
- Agent SDK read the compiled manifest and resolved selector scope.

## Review Friction

- `--approve-defaults` is usable for the fixture, but real protocol review still needs operator attention for out-of-scope selectors and human-authored intent.
- The review flow depends on draft metadata input mappings for manifest contract IDs to analysis contract IDs.

## Schema Friction

- The fixture maps cleanly to `scope.contracts`, `roles`, and all three invariant classes.
- BattleChain linkage is clean as adapter metadata and does not need core manifest fields.

## Default / Template Quality

- Ownable, Pausable, and Upgradeable recognitions are useful for the fixture.
- The fixture is too small to prove ERC4626 or complex multi-contract mapping quality.

## Suggested Fixes

- Keep the SimpleVault fixture as the release smoke path.
- Add external protocol dogfood reports before broadening recognizer behavior.
- Keep registry publish as a prepared payload until signer and authorization requirements are explicit.
